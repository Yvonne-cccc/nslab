# transparent web proxy 

We use squid as our transparent web proxy. Squid will run on the same computer as the router. To be transparent we have to redirect web traffic to squid without knowledge of the clients. We do this via iptables on the router:

HTTP:
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 3128

HTTPS:
iptables -t nat -A PREROUTING -p tcp --dport 443 -j REDIRECT --to-port 3129


Because HTTPS is encrypted Squid doesnt decode the HTTPS traffic, but can filter for the host.


Squid needs to take the redirected traffic and handle it transparently. This can be configured in squid.conf:

http_port 3127																				#used for the webinterface of squid

http_port 3128 intercept																	#handle incoming packages on 3128 transparently
 
acl step1 at_step SslBump1																	#step1 is TRUE if there is TLS (HTTPS)
ssl_bump peek step1 																		#"peek" into package if step1 is TRUE
ssl_bump splice all																			#dont encrypt the package (to stay transparent) and send it further to its destination
https_port 3129 intercept ssl-bump cert=/etc/squid/ca.pem generate-host-certificates=on		#listen to port=3129 (squid needs a private key and certificate in ca.pem to sign certificates for HTTPS connections)

"sslcrtd_program /usr/lib/squid/security_file_certgen -s /var/spool/squid/ssl_db -M 4MB" needs to be included for squid to be able to use ssl_bump


The ca.pem file gets automaticly generated by executing "ca_pem_maker.sh".

In the terminal "sudo chown -R proxy:proxy /var/spool/squid/ssl_db" allows squid to access its SSL certificate database where it stores signed certificates of HTTPS connections
